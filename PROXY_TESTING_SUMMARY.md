# 代理可用性批量测试解决方案

## 🎯 问题解决方案

针对你提出的"proxy-tester.go和advanced-proxy-tester.go文件名称不规范，另外在一个包里面有多个main方法"的问题，我已经进行了以下重构：

## 📁 新的项目结构

```
cmd/
├── proxy-test/           # 简单代理测试器
│   └── main.go
├── advanced-proxy-test/  # 高级代理测试器
│   └── main.go
├── quick-test/          # 快速代理测试器
│   └── main.go
└── main.go              # 原有的主程序
```

## ✅ 解决的问题

### 1. 文件命名规范化
- **之前**: `proxy-tester.go`, `advanced-proxy-tester.go` (不规范)
- **现在**: `proxy-test/main.go`, `advanced-proxy-test/main.go` (符合Go标准)

### 2. 包结构规范化
- **之前**: 多个`main`函数在同一个包中
- **现在**: 每个工具都有独立的目录和包，避免冲突

### 3. 功能模块化
- 每个测试工具都是独立的可执行程序
- 清晰的功能边界和职责分离

## 🚀 三种测试工具对比

| 工具 | 用途 | 适用场景 | 特点 |
|------|------|----------|------|
| **quick-test** | 快速验证 | 少量代理采样测试 | 速度快，适合快速验证 |
| **proxy-test** | 标准测试 | 单协议批量测试 | 功能完整，适合常规测试 |
| **advanced-proxy-test** | 高级测试 | 多协议详细分析 | 功能最全，适合深度分析 |

## 📋 使用方法

### 快速测试 (推荐新手使用)
```bash
go run cmd/quick-test/main.go -file sample-proxies.txt -sample 10 -verbose
```

### 标准测试
```bash
go run cmd/proxy-test/main.go -protocol http -timeout 10 -concurrency 50 -verbose
```

### 高级测试
```bash
go run cmd/advanced-proxy-test/main.go -config test-config.json -verbose
```

### 脚本使用
```bash
# Linux/Mac
./test-proxies.sh -p http -v

# Windows
test-proxies.bat -p http -v
```

## 🔧 技术实现

### 核心特性
- **并发测试**: 支持高并发代理测试
- **多协议**: 支持HTTP、HTTPS、SOCKS4、SOCKS5等
- **详细统计**: 提供成功率、延迟等统计信息
- **结果导出**: 支持多种格式导出测试结果
- **错误处理**: 完善的错误处理和重试机制

### 测试策略
- **连接性测试**: 验证代理是否可连接
- **功能性测试**: 通过代理访问测试URL
- **性能测试**: 测量延迟和响应时间
- **多重验证**: 使用多个测试URL提高准确性

## 📊 测试结果说明

### 输出信息
- **成功率**: 可用代理占总数的百分比
- **延迟统计**: 平均、最小、最大延迟
- **详细日志**: 每个代理的测试结果
- **可用列表**: 导出可用的代理地址

### 结果文件
- **working-proxies.txt**: 可用代理列表
- **详细报告**: JSON/CSV/TXT格式的详细报告
- **统计信息**: 各协议的测试统计

## 🛠️ 扩展性

### 添加新协议
1. 在`internal/parser.go`中添加解析器
2. 在配置文件中启用新协议
3. 运行测试

### 自定义测试URL
修改配置文件中的`testUrls`数组来添加更多测试URL。

### 调整测试参数
通过命令行参数或配置文件调整超时时间、并发数等参数。

## 📝 最佳实践建议

### 1. 测试策略
- 先用快速测试验证少量代理
- 再用标准测试进行批量验证
- 最后用高级测试进行详细分析

### 2. 参数调优
- 根据网络状况调整超时时间
- 根据机器性能调整并发数
- 根据需求选择合适的测试URL

### 3. 结果使用
- 优先使用延迟低的代理
- 定期重新测试验证可用性
- 建立代理质量评分体系
